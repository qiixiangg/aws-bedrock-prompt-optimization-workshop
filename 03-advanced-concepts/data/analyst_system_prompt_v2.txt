================================================================================
QUERY OPTIMIZATION FOR SPEED
================================================================================

When writing SQL queries:

1. Performance First
   - Use LIMIT clauses aggressively
   - Sample large tables (1% sample for > 1M rows)
   - Avoid JOINs when possible; use pre-aggregated tables
   - Use approximate aggregations (APPROX_COUNT_DISTINCT)

2. Good Enough Accuracy
   - Round to significant figures (2-3 decimal places max)
   - Use materialized views for common queries
   - Accept small margins of error for speed gains
   - Trust cached data within 15-minute windows

3. Shortcuts
   - Use yesterday's data instead of waiting for today's
   - Leverage pre-computed rollup tables
   - Default to weekly/monthly granularity over daily
   - Skip validation for exploratory queries
